Mark concept
Three ascending bars encode benchmark data. Left + center bars in white (opacity-stepped), right bar in accent green — the "positive result" signal. Baseline rule grounds the chart. Works at 16px (favicon) through 512px (PWA).
02 — Wordmark (Stacked, Light Background)
Stacked — light bg
Typography rationale
DM Sans throughout — not DM Serif Display, which is reserved for page H1s. A logo must be legible at 14px in a nav bar; display serifs fail at that size.

"MainStreet" semibold (#1B2B4B) — the qualifier. Navy matches the nav background.

"Benchmark" bold (#1E6B45) — the differentiator. Green encodes the same positive data result signal used for the valuation output. Two-weight split creates emphasis without a second typeface.

07 — Complete <head> Favicon Implementation
<!-- Favicon implementation — copy exactly into every page <head> -->

<!-- Legacy ICO: covers IE11, old Edge, Windows pinned sites -->
<link rel="icon" href="/favicon.ico" sizes="32x32">

<!-- SVG: modern browsers (Chrome 80+, Firefox 72+, Edge 80+) -->
<!-- Scales perfectly at any resolution, smallest file -->
<link rel="icon" href="/favicon.svg" type="image/svg+xml">

<!-- Apple Touch Icon: iOS Safari home screen, iMessage link previews -->
<!-- No rounded corners in file — iOS adds them automatically -->
<link rel="apple-touch-icon" href="/apple-touch-icon.png">

<!-- PWA Manifest: Android home screen, Chrome install prompt -->
<link rel="manifest" href="/manifest.webmanifest">

<!-- Theme color: Chrome mobile address bar, Windows taskbar -->
<meta name="theme-color" content="#1B2B4B">

<!-- Force light mode: overrides OS dark mode -->
<!-- Per 12-VISUAL-DESIGN-SPEC.md: no dark mode implementation -->
<meta name="color-scheme" content="light">
